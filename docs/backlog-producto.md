# Backlog de producto

## PBIs priorizados

| ID | Historia de usuario | Prioridad | Criterios de aceptación |
| --- | --- | --- | --- |
| PBI-01 | Como candidato, quiero registrarme para iniciar el proceso de uso del sistema. | Alta | Se crea usuario con nombre/correo válido; evita correos duplicados. |
| PBI-02 | Como candidato, quiero cargar texto de CV para extraer habilidades iniciales. | Alta | El sistema detecta habilidades clave y crea/actualiza perfil. |
| PBI-03 | Como candidato, quiero editar mi perfil para corregir habilidades y experiencia. | Alta | El perfil queda persistido y se refleja en recomendaciones. |
| PBI-04 | Como candidato, quiero ver vacantes ordenadas por compatibilidad. | Alta | Se muestran score y motivo de match por vacante. |
| PBI-05 | Como candidato, quiero postular manualmente a una vacante. | Alta | Se crea postulación y no permite duplicados para la misma vacante. |
| PBI-06 | Como candidato, quiero postulación asistida para aplicar más rápido. | Media | El sistema crea varias postulaciones sobre recomendaciones top. |
| PBI-07 | Como candidato, quiero configurar postulación automática por filtros. | Media | Se guarda configuración (mín, máx, frecuencia, keyword, ubicación). |
| PBI-08 | Como candidato, quiero ver seguimiento de estados de mis postulaciones. | Alta | Permite visualizar y actualizar estado: Postulado, En revisión, Entrevista, Rechazado. |
| PBI-09 | Como candidato, quiero ver un home con métricas de progreso. | Media | Muestra total, progreso y conteo por estado con recomendaciones destacadas. |
